# code-music

Code-generated music: write Python blocks, export Spotify-ready audio.

**Audience docs** (these are the primary user-facing entry points):
- `docs/for_listeners.md` — just press play, no code
- `docs/for_creators.md` — make your first song in 5 minutes
- `docs/for_theory_heads.md` — full reference: scales, ornaments, effects
- `docs/for_developers.md` — architecture, extension points, CI

**Beginner template:** `songs/_template_beginner.py`

## Quick start (developer)

```bash
make dev          # create .venv, install deps, check ffmpeg
make test         # run all tests
make songs-flac   # render all songs → dist/flac/ (Spotify-ready)
make songs-all    # WAV + FLAC + MP3 for all songs
make samples      # render all sample previews → dist/samples/
make help         # full target list
```

Requires Python 3.11+. FLAC/MP3/OGG export requires ffmpeg (`brew install ffmpeg`).

## Repo layout

```

  engine.py         Note, Chord, Beat, Track, Song, scale()
  synth.py          Synth — renders Song → stereo float64 numpy array
  effects.py        reverb, delay, chorus, distortion, filters, compress, pan
  export.py         export_wav / export_flac / export_mp3 / export_ogg
  cli.py            code-music <script.py> [--wav|--flac|--mp3|--ogg] [-o path]

songs/              Full song compositions (each defines a `song` variable)
samples/
  bass/             Low-end bass patches
  highs/            High-frequency leads and arpeggios
  synths/           Pad, organ, lead synth demos
  waves/            Raw waveform comparisons (sine, saw, triangle, square)
  chords/           Chord progression demos
  instruments/      Piano, plucked guitar, etc.
  drums/            Kick, snare, hat patterns

tests/              pytest suite
dist/               Rendered audio output — gitignored, generated by `make`
  wav/              Lossless PCM
  flac/             Lossless compressed (preferred for Spotify)
  mp3/              Lossy 320kbps
  samples/          Short sample previews
```

## Writing a song

A song script must define a top-level `song` variable:

```python
from code_music import Note, Chord, Track, Song, scale, reverb, delay

song = Song(title="My Track", bpm=128)

pad = song.add_track(Track(name="pad", instrument="pad", volume=0.5, pan=-0.2))
pad.add(Chord("A", "min7", 3, duration=8.0))

lead = song.add_track(Track(name="lead", instrument="sawtooth", volume=0.6, pan=0.3, swing=0.0))
lead.extend(scale("A", "pentatonic", octave=5, length=8))

# Per-track effects — applied during render
song._effects = {
    "pad":  lambda s, sr: reverb(s, sr, room_size=0.7, wet=0.3),
    "lead": lambda s, sr: delay(s, sr, delay_ms=375.0, feedback=0.3, wet=0.25),
}
```

Render it:
```bash
code-music songs/my_track.py --flac          # → dist/flac/my_track.flac
code-music songs/my_track.py --mp3           # → dist/mp3/my_track.mp3
code-music songs/my_track.py -o /tmp/out.wav # custom path
```

## Instruments

| Name           | Character                          |
|----------------|------------------------------------|
| `sine`         | Pure, clean                        |
| `square`       | Hollow, 8-bit buzzy                |
| `sawtooth`     | Bright, cutting, supersaw-ready    |
| `triangle`     | Soft, flute-like                   |
| `piano`        | Short attack, natural decay        |
| `organ`        | Sustained, Hammond-ish             |
| `bass`         | Low sawtooth, punchy               |
| `pad`          | Slow attack, lush ambient          |
| `pluck`        | Sharp attack, Karplus-Strong feel  |
| `drums_kick`   | Pitch-dropping sine kick           |
| `drums_snare`  | Noise + tone snare                 |
| `drums_hat`    | High-frequency hat                 |

## Effects

All take `(samples: ndarray, sample_rate: int, **kwargs)` → `ndarray`.

| Function      | Purpose                                   |
|---------------|-------------------------------------------|
| `reverb`      | Schroeder reverb (room_size, damping, wet)|
| `delay`       | Stereo echo with ping-pong option         |
| `chorus`      | LFO-modulated delay                       |
| `distortion`  | Soft-clip overdrive                       |
| `lowpass`     | Biquad LP filter (cutoff_hz, q)           |
| `highpass`    | Biquad HP filter                          |
| `bandpass`    | Biquad BP filter                          |
| `compress`    | Peak-following compressor                 |
| `pan`         | Equal-power stereo pan (-1..1)            |

## Spotify pipeline

1. `make spotify` → renders to `dist/flac/` and prints upload URL
2. Spotify for Artists → Music → Upload Track
3. Min requirements: 30s, 44100 Hz stereo
4. FLAC preferred; MP3 320k also accepted

## Conventions

- One `song` variable per file in `songs/`.
- Sample files are short (2–8 bars) instrument/pattern demos.
- `dist/` is gitignored — never commit rendered audio.
- Tests use `sample_rate=22050` for speed; production uses `44100`.
- Per-track effects live in `song._effects` dict (name → callable).
